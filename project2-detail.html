Project 2: Incompleteness

Color Arcade

1. The Concept

This project explores Incompleteness through the mathematical concepts of Congruence versus Similarity. Here, completeness is congruence while incompleteness is similarity, a continuous journey of striving toward an ideal rather than a failure.

2. The Outcome

Color Arcade is an interactive web experience that challenges the imprecision of the human eye. Wrapped in a pixelated arcade aesthetic, it offers players a playful way to test their color perception.

3. Technical Execution

The core of Project 2 lies in translating abstract human perception into data. Below are the three key technical systems that power the experience: scoring logic, dynamic difficulty scaling, and a unified data management system.

A. Color: Similarity Scoring

Since the Hue in HSL is circular (0–360°), the code calculates the shortest distance between two points on a wheel.

let hDiff = Math.abs(targetH - uH);
if (hDiff > 180) hDiff = 360 - hDiff;
const hScore = 100 - (hDiff / 180 * 100);

D. Action Logic: Paint Drop Mixing

In the Paint Drop mode, the code calculates the final color by averaging the RGB values of all added drops. The similarity score is determined using the 3D Euclidean distance formula within the RGB color space.

const distance = Math.sqrt(
    Math.pow(targetR - uR, 2) + 
    Math.pow(targetG - uG, 2) + 
    Math.pow(targetB - uB, 2)
);
const maxDist = 441.67; 
const score = Math.max(0, 100 - (distance / maxDist * 100));
